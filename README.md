# Client Renewal Health Intelligence

A lightweight end-to-end prototype for surfacing renewal risk signals from client communications at a community bank.

Built as an Applied AI / AI Product Analyst take-home assessment prototype.

---

## What this prototype does

**Step 1 (current):** Generates realistic sample client communication data and ingests it into a unified, structured dataset ready for analysis.

The pipeline covers four raw communication channels for a fictional client — Harborview Community Bank — and normalizes them into a single `client_health_records.csv`.

Planned next steps (not yet implemented):
- Step 2: Streamlit dashboard to visualize client health and renewal-risk signals
- Step 3: LLM-based classification and automated renewal readiness brief generation

## Setup

### 1. Create and activate a virtual environment

```bash
python -m venv .venv
source .venv/bin/activate      # macOS / Linux
# .venv\Scripts\activate       # Windows
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

---

## Running the pipeline

### Step 1 — Generate sample data

Creates four raw CSV files in `data/raw/`:

```bash
python src/generate_sample_data.py
```

### Step 2 — Run the ingestion pipeline

Reads the raw CSVs, normalizes them into one unified schema, and writes:
`data/processed/client_health_records.csv`

```bash
python src/ingest.py
```

---

## Unified schema

| Column | Description |
|---|---|
| `record_id` | Unique ID per record |
| `date` | Parsed date of the communication |
| `source` | Normalized source type (e.g. `support_ticket`, `account_email`) |
| `client_name` | Client organization name |
| `contact_role` | Role of the contact who authored the communication |
| `product_area` | Platform area referenced |
| `raw_text` | Cleaned communication text |
| `status` | Status or resolution state |
| `priority` | Priority or urgency level |
| `owner` | Nymbus account team member responsible |
| `original_channel` | Source channel label (e.g. `Support Portal`, `Email`) |

## Sample client

All data represents a single fictional client:

- **Client:** Harborview Community Bank
- **Tenure:** 3 years on platform
- **Renewal:** Due in ~4 months
- **Signals:** Mix of unresolved tickets, escalating emails, QBR commitments, and negative product feedback
